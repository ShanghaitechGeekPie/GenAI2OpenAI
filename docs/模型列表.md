### 上下文长度测试说明

**测试方法**：使用大海捞针法（Needle in a Haystack），在文本中间（50%深度）插入关键信息，测试模型能否准确检索。

**测试结果详情**：

- **deepseek-r1**: 100k tokens 通过，150k tokens 失败（返回空响应）
- **deepseek-v3**: 100k-200k tokens 全部通过（至少支持 200k）
- **glm-5.1**: 100k-200k tokens 全部通过（至少支持 200k）
- **minimax-m1**: 100k-200k tokens 全部通过（至少支持 200k）
- **qwen3.5-397b-a17b**: 100k tokens 即返回 HTTP 500 错误
- **GPT 系列**: 未测试。Azure OpenAI 模型有严格的额度限制，测试中所有请求返回空响应（`content: ""`），无法完成上下文长度测试。根据官方文档，GPT-4.1 支持 1M tokens，GPT-5/o3 等支持 128k-256k tokens。

**注意**：

1. 实测上下文长度可能受上游服务稳定性影响
2. GPT 系列模型因 Azure 额度限制未进行上下文测试，建议参考官方文档了解其标称上下文长度
3. 建议使用前先用小文本验证目标模型的实际表现
4. 当前部署的 access token 可能对 GPT 模型有额外限制
